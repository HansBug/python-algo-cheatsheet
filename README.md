## 📖 这是什么？

这是一个专注于**求职笔试和面试**的 Python 算法模板集合。所有代码都是为了**让你背下来**而设计的，不是让你当第三方库调用的。

### 核心特点

- ✅ **发挥 Python 优势**：充分利用 Python 语言特性（列表推导、切片、内置函数等）
- ✅ **规避 Python 劣势**：针对性能瓶颈优化，避免常见的速度陷阱
- ✅ **简短易记**：代码精简，方便理解和记忆，考场上能快速写出
- ✅ **兼容性强**：纯原生库实现，支持 Python 3.7-3.14，保证 OJ 平台兼容
- ✅ **实战导向**：所有模板都经过实际题目验证

## 🎯 适合谁？

## 🧠 复杂性分数怎么看？

这些分数由 `python -m tools.interview_complexity` 对各算法**推荐方案的核心代码**测得。标准库 / 原生容器方案按最小可用模板片段测，手写数据结构方案按对应实现类测。

- `0-6`：**很容易手写**。基本就是 API 或容器套路，几分钟能写完，真正的难点通常在题目本身。
- `6-12`：**正常可写**。需要记住少量不变量和边界，熟练后可以稳定写出来。
- `12-18`：**坑比较多**。分支、状态和边界明显变多，没专门练过的话，现场容易开始 debug。
- `18+`：**现场高风险**。模板本身就有明显手写负担，建议提前刻意训练，不要指望面试时现推。

分数**没有上限**。同样都叫“现场高风险”，`20` 分、`60` 分、`150` 分的体感差别其实很大：分数越高，通常意味着代码更长、分支更多、状态/指针维护更多。
