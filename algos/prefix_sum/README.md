# 前缀和笔试攻略

## 什么是前缀和？

前缀和就是预处理出“前 i 个元素的累计和”，这样区间和查询就能从 **O(n)** 降到 **O(1)**。

- 常见写法是 `pre[0] = 0`
- `pre[i]` 表示前 `i` 个元素之和
- 闭区间 `[l, r]` 的和是 `pre[r + 1] - pre[l]`
- 如果题目变成“子数组和满足某条件”，通常要配合 **哈希表**

### 支持的操作

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 建前缀和 | O(n) | 扫一遍数组 |
| 查询区间和 | O(1) | `pre[r + 1] - pre[l]` |
| 子数组和计数 | O(n) | 前缀和 + 哈希表 |
| 二维区域和 | O(1) 查询 | 先 O(mn) 预处理 |

## 前缀和适合解决什么问题？

1. **区间和查询**
    - 例如：多次查询某段数组之和
    - 时间复杂度：预处理 O(n)，单次查询 O(1)

2. **子数组和等于 K**
    - 例如：统计满足条件的连续子数组个数
    - 时间复杂度：O(n)，优于 O(n^2) 枚举

3. **0/1 平衡、正负平衡问题**
    - 例如：0 和 1 数量相同的最长子数组
    - 时间复杂度：O(n)

4. **矩阵区域和**
    - 例如：二维前缀和查询子矩阵和
    - 时间复杂度：预处理 O(mn)，查询 O(1)

## 笔试场景建议

### 推荐策略

1. **只要出现“多次区间和查询”，优先想前缀和**
    - ✅ 套路固定
    - ✅ 代码短
    - ✅ 面试里很高频

2. **子数组计数题优先想“前缀和差值”**
    - ✅ `pre[j] - pre[i] = k` 就是在找某个历史前缀和
    - ✅ 往往能接哈希表做 O(n)

3. **统一索引语义**
    - ✅ 一律让 `pre[i]` 表示前 i 个元素
    - ❌ 不要一会儿按长度，一会儿按原数组下标

## 方案一：通用模板（推荐）

### 一维前缀和

```python
pre = [0]
for x in nums:
    pre.append(pre[-1] + x)
```

### 区间和查询

```python
def range_sum(pre, left, right):
    return pre[right + 1] - pre[left]
```

### 子数组和等于 K

```python
cnt = {0: 1}
s = 0
ans = 0

for x in nums:
    s += x
    ans += cnt.get(s - k, 0)
    cnt[s] = cnt.get(s, 0) + 1
```

## 常见题型速查

### 1. 多次区间求和

```python
pre = [0]
for x in nums:
    pre.append(pre[-1] + x)

for left, right in queries:
    ans = pre[right + 1] - pre[left]
```

### 2. 和为 K 的子数组个数

```python
cnt = {0: 1}
s = 0
ans = 0

for x in nums:
    s += x
    ans += cnt.get(s - k, 0)
    cnt[s] = cnt.get(s, 0) + 1
```

### 3. 0 和 1 数量相同的最长子数组

```python
first = {0: -1}
s = 0
ans = 0

for i, x in enumerate(nums):
    s += 1 if x == 1 else -1
    if s in first:
        ans = max(ans, i - first[s])
    else:
        first[s] = i
```

## 实战技巧总结

✅ **前缀和数组多开一位 0**：最省脑子

✅ **闭区间统一写 `pre[r + 1] - pre[l]`**

✅ **子数组计数题优先想哈希表记历史前缀和**

✅ **平衡题常把两种状态映射成 `+1 / -1`**

✅ **二维题先写清楚索引定义**：别在加减号里绕晕

✅ **查询多、修改少时优先用前缀和**
